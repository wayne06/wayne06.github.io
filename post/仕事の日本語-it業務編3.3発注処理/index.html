仕事の日本語－3．仕様書を読む－文章表現に慣れておこう－3．発注処理

0. フローチャート

  • 0.　スタート

  • 1.　在庫移動トップ（店舗の在庫数が発注点を下回った時は、在庫移動画面が表示される）

  • 2.　商品コード、数量、移動先を入力（ユーザーが入力した商品コードと数量によって、移動元の倉庫の在庫引当を行う。）

  • 3.　発注点チェック

    • if 発注点以下

      • 3.1　在庫補充（移動によって、移動元の倉庫の在庫数が発注点以下になる時は、在庫補充画面が開かれる。）

      • 3.2　仕入先チェック

        • if 仕入先が複数（補充する商品の仕入先が複数する時は、仕入先選択画面を表示する。）　⇒　3.3

        • if 仕入先が一つ　⇒　3.4

      • 3.3　仕入先選択　⇒　3.4

      • 3.4　発注確認　⇒　4

    • if 発注点より多い（倉庫の在庫数が発注点よりも多い時は、在庫移動を確定する。）　⇒　4

  • 4.　在庫移動確定　⇒　5

  • 5.　エンド

1. 目標

  • 仕様書の在庫管理に関する語彙が理解できる

  • 在庫移動処理の流れが理解できる

2. 表現の理解

  • 店舗の在庫数が発注点を下回った時は、在庫移動画面が表示される。

  • ユーザーが入力した商品コードと数量によって、移動元の倉庫の在庫引当を行う。

  • 倉庫の在庫数が発注点よりも多い時は、在庫移動を確定する。

  • 移動によって、移動元の倉庫の在庫数が発注点以下になる時は、在庫補充画面が開かれる。

  • 補充する商品の仕入先が複数する時は、仕入先選択画面を表示する。
